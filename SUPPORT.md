# Support

Thank you for engaging with Karma Infinity.

This file helps contributors, users, partners, and community members find the right pathway for help.

## General questions

For general project questions, use GitHub Discussions if enabled on the relevant repository.

If Discussions are not enabled, open an issue using the closest matching issue template.

## Bug reports

For bugs, open an issue using the bug report template and include:

- What happened
- What you expected
- Steps to reproduce
- Screenshots or logs, if useful
- Environment details
- Accessibility or safety impact, if any

## Feature ideas

For feature ideas, use the feature request template and explain:

- The problem
- Who benefits
- Why it matters
- Possible risks or trade-offs
- How it aligns with Karma Infinity’s mission

## Documentation help

For documentation improvements, use the documentation template.

Good documentation issues identify what is unclear, missing, outdated, or hard to follow.

## Governance, ethics, and safety concerns

For governance, ethics, accessibility, community safety, privacy, bias, or mission-alignment concerns, use the governance and ethics issue template unless the matter is private or sensitive.

For private concerns, contact maintainers directly through the appropriate private channel.

## Security vulnerabilities

Do not open a public issue for security vulnerabilities.

Follow `SECURITY.md`.

## Funding and partnership inquiries

Funding and partnership pathways will be listed in `FUNDING.yml` and future official Karma Infinity channels.

Do not send sensitive financial, legal, personal, or partnership information through public GitHub issues.

## Response priorities

Karma Infinity is early-stage. Maintainer response times may vary.

We aim to prioritize:

1. Security and safety issues
2. Broken functionality
3. Accessibility barriers
4. Documentation blockers
5. Mission-aligned feature proposals
6. General questions

## Community standard

All support interactions must follow `CODE_OF_CONDUCT.md`.
