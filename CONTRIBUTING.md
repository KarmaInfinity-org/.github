# Contributing to Karma Infinity

Thank you for your interest in contributing to Karma Infinity.

This ecosystem is being built to serve people with dignity, care, safety, accessibility, and long-term responsibility. Contributions are welcome when they strengthen the mission and respect the standards below.

## Contribution principles

Before contributing, align with these principles:

1. Build for human benefit
2. Protect privacy and safety
3. Keep accessibility in mind from the start
4. Prefer clarity over cleverness
5. Document decisions and trade-offs
6. Avoid hype, unsupported claims, and unnecessary complexity
7. Respect maintainers, contributors, users, and affected communities
8. Leave the system better than you found it

## Ways to contribute

You can contribute by:

- Reporting bugs
- Suggesting features
- Improving documentation
- Reviewing accessibility issues
- Proposing ethical safeguards
- Improving tests and reliability
- Helping design impact measurement
- Reviewing security and privacy risks
- Creating educational resources
- Supporting governance and community standards

## Before opening an issue

Please check whether:

- A similar issue already exists
- The request belongs in a specific repository instead of the organization-level `.github` repository
- The topic involves a security vulnerability, in which case you must follow `SECURITY.md`
- The idea is aligned with Karma Infinity’s mission and values

## Issue quality standards

A strong issue includes:

- Clear title
- Problem statement
- Expected outcome
- Relevant context
- Steps to reproduce, if reporting a bug
- Screenshots or logs, if helpful
- Accessibility, privacy, safety, or ethical considerations
- Suggested solution, if known

## Pull request expectations

Before opening a pull request:

- Keep the change focused
- Explain what changed and why
- Link related issues
- Add or update documentation when needed
- Add tests when applicable
- Check formatting and spelling
- Avoid unrelated cleanup in the same pull request
- Confirm that no secrets, credentials, personal data, or private information are included

## Commit guidance

Use clear commit messages.

Recommended style:

```text
type(scope): short description
```

Examples:

```text
docs(profile): clarify Karma Infinity mission statement
chore(templates): add governance issue template
security(policy): update disclosure process
```

Common types:

- `docs`
- `feat`
- `fix`
- `chore`
- `refactor`
- `test`
- `security`
- `governance`

## Branch naming

Recommended branch format:

```text
type/short-description
```

Examples:

```text
docs/update-support-policy
feat/add-impact-arm-template
security/improve-reporting-language
```

## Review standards

Maintainers may ask for changes related to:

- Mission alignment
- Security and privacy
- Accessibility
- Documentation clarity
- Maintainability
- Evidence quality
- Ethical risk
- Scope control

Review is not personal. It is part of protecting the quality and trustworthiness of the ecosystem.

## AI-assisted contributions

AI-assisted work is allowed when used responsibly.

If AI tools materially contributed to a pull request, disclose that in the pull request description and confirm that you reviewed the output for:

- Accuracy
- Security risks
- License issues
- Privacy concerns
- Hallucinated references
- Bias or harmful assumptions
- Accessibility problems

Contributors remain responsible for what they submit.

## Security and privacy

Never include:

- API keys
- Passwords
- Private tokens
- Personal information
- Private user data
- Internal credentials
- Unredacted logs with sensitive data

Security vulnerabilities must be reported through `SECURITY.md`, not public issues.

## Accessibility baseline

When contributing user-facing content or interfaces, consider:

- Clear language
- Keyboard navigation
- Screen reader compatibility
- Color contrast
- Mobile readability
- Captions or alternatives for media
- Avoiding unnecessary motion

## Mission fit check

Before submitting, ask:

- Does this reduce harm or increase usefulness?
- Does this protect dignity?
- Could this create privacy, safety, or accessibility risks?
- Is this understandable to future contributors?
- Is this aligned with a neutral, pro-human mission?

## Contributor license note

By contributing, you agree that your contribution may be included under the license of the repository where it is submitted.

## Thank you

Karma Infinity is built through disciplined compassion: practical work, ethical standards, and a shared commitment to helping people.
